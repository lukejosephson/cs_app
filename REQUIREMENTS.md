#Product Requirements Document

**Features**

Necessary for functionality: 


1. [ ] Authentication: User opens the app to a sign in / create account screen

2. [ ] UI: User is shown a home screen that gives the option of puzzles / types of problems to solve 

3. [ ] Binary Practice: User should be given a number and asked to flip tiles from 0 to 1 
to make the number, which should then be verified

4. [ ] Error Detection: User should be given a code snippet with a syntax or logic error.
User should be able to select where the error occurs.

5. [ ] Loop Tracing: User should be given a code snippet with a loop that modifies variables,
and be prompted to trace the final result of one of the variables

6. [ ] User should be shown fresh questions each time

7. [ ] User should have the option to retry questions they got wrong later on

Future Possibilities:

10. [ ] User can access profile that shows history of practice and allows 
them to view their stats (number attempted and solved) for each type of practice.

11. [ ] App keeps track of days with a type of puzzle completed in a row
and includes achievements for specific milestones / puzzles solved

12. [ ] Maybe: user can opt for any coding language for puzzles


### Developer Responsibilities

1. Test each feature after it is added

2. Make sure a test was added to the test directory

3. Review the code and make sure you understand it.

4. Code architecture. Make sure that responsibilities are in seperate files and
that widgets don't get too large. If the do, put them in seperate files. Make sure
data models are clearly seperated from the UI.
